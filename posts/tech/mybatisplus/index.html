  • public interface UserMapper extends BaseMapper<User> {
}

  • 继承的原因是为了使用MP中已经定义好的方法

  • 需要注意的是，mybatis与mybatis-plus配置结果集映射到对象的不同之处

    • mybatis-plus

      • 在继承BaseMapper中指定泛型来配置映射关系，如上述

    • mybatis

      • xml方式：resultType【结果集】配置映射关系
      • 注解方式：在mapper接口中的方法增加注解，将方法返回值的泛型配置映射关系，如：public List<User> findAll();
✅引入mybatis-plus依赖#

  • 将SqlSessionFactoryBuilder类改成MybatisSqlSessionFactoryBuilder类

  • //SqlSessionFactory sqlSessionFactory = new SqlSessionFactoryBuilder().build(inputStream);
SqlSessionFactory sqlSessionFactory = new MybatisSqlSessionFactoryBuilder().build(inputStream);

  • 作用

    • 将mybatis-plus依赖中集成的sql语句引入mybatis框架【实际上MybatisSqlSessionFactoryBuilder继承并重写了SqlSessionFactoryBuilder】
✅指定表【配置映射关系之指定表】#
  • mybatis原生操作时，无论是xml还是注解，都会使用具体的sql语句，所以通过sql语句就可以知道操作是的哪张表
  • mybatis-plus与mybatis不同的是【此处类的名字为：类的首字母小写之后的名字】
    • 如果表的名字与类的名字一致
      • mybatis-plus可以自动映射关系
    • 如果表的名字与类的名字不一致
      • 需要将类指定表，在类上增加注解，格式如：@TableName("tb_user")

⭐插入操作#

✅默认规则#
  • 插入的时候，根据驼峰命名，类的属性与表的字段自动匹配
  • 如果插入时，对象的某个属性值为空，则表的字段值会设置为空
    • 但是字段设置了非空约束，属性就必须有值，否则会报错
✅@TableId#

  • 作用：自动生成id值

  • 使用步骤：在类的id属性上增加@TableId注解，当插入数据成功时，会自动设置id值

  • 根据注解@TableId不同的枚举值，id的生成策略如下

    public @interface TableId {

    /**
     * 字段值（驼峰命名方式，该值可无）
     */
    String value() default "";

    /**
     * 主键ID
     * {@link IdType}
     */
    IdType type() default IdType.NONE;
}

  • 修改完id生成策略后，还需要在数据库修改表的自动增量

✅@TableField#

  • 作用

    • 对象中的属性名和字段名不一致，将属性指定字段，格式如：@TableField(value = "email")
    • 对象中的属性字段在表中不存在，忽略这个属性，格式如：@TableField(exist = false)
    • 指定某个属性不加入 select 查询范围，格式如：@TableField(select = false)

  • 具体API如下

    public @interface TableField {

    /**
     * 字段值（驼峰命名方式，该值可无）
     */
    String value() default "";

    /**
     * 当该Field为类对象时, 可使用#{对象.属性}来映射到数据表.
     * <p>支持：@TableField(el = "role, jdbcType=BIGINT)</p>
     * <p>支持：@TableField(el = "role, typeHandler=com.baomidou.springcloud.typehandler.PhoneTypeHandler")</p>
     */
    String el() default "";

    /**
     * 是否为数据库表字段
     * <p>默认 true 存在，false 不存在</p>
     */
    boolean exist() default true;

    /**
     * 字段 where 实体查询比较条件
     * <p>默认 `=` 等值</p>
     */
    String condition() default "";

    /**
     * 字段 update set 部分注入, 该注解优于 el 注解使用
     * <p>例如：@TableField(.. , update="%s+1") 其中 %s 会填充为字段</p>
     * <p>输出 SQL 为：update 表 set 字段=字段+1 where ...</p>
     * <p>例如：@TableField(.. , update="now()") 使用数据库时间</p>
     * <p>输出 SQL 为：update 表 set 字段=now() where ...</p>
     */
    String update() default "";

    /**
     * 字段验证策略
     * <p>默认追随全局配置</p>
     */
    FieldStrategy strategy() default FieldStrategy.DEFAULT;

    /**
     * 字段自动填充策略
     */
    FieldFill fill() default FieldFill.DEFAULT;

    /**
     * 是否进行 select 查询
     * <p>大字段可设置为 false 不加入 select 查询范围</p>
     */
    boolean select() default true;

    /**
     * 是否保持使用全局的 Format 的值
     * <p> 只生效于 既设置了全局的 Format 也设置了上面 {@link #value()} 的值 </p>
     * <li> 如果是 false , 全局的 Format 不生效 </li>

✅BiPredicate<R, V>#
  • 一个函数式接口，它接受两个参数并返回一个boolean类型，通常在集合对象的筛选器中使用。
  • 原理：将集合中的元素满足表达式时，继续留在集合中。当元素不满足表达式时，则剔除在集合中。但该集合在内存中的数据不会发生变化，只是该集合在作为参数传递时，传递的数据发生了变化【即不满足的元素不会被作为数据传递】

    /**
     * SELECT id,user_name,name,age,email AS mail FROM tb_user WHERE password IS NULL AND name = ? AND age = ?

ActiveRecord#

1️⃣概述#

  • ✔Active Record(活动记录)，是一种领域模型模式，特点是一个模型类对应关系型数据库中的一个表，而模型类的一个实例对应表中的一行记录

  • ✔ActiveRecord也属于ORM（对象关系映射）层，由Rails最早提出，遵循标准的ORM模型

    • 表映射到记录，记录映射到对象，字段映射到对象属性。
    • 配合遵循的命名和配置惯例，能够很大程度的快速实现模型的操作，而且简洁易懂。

  • ✔ActiveRecord的主要思想是：

    • 每一个数据库表对应创建一个类，类的每一个对象实例对应于数据库中表的一行记录；通常表的每个字段在类中都有相应的Field
    • ActiveRecord同时负责把自己持久化，在ActiveRecord中封装了对数据库的访问，即CURD
    • ActiveRecord是一种领域模型(Domain Model)，封装了部分业务逻辑

1️⃣mybatis的插件机制#

✅概述#
  • MyBatis 允许你在已映射语句执行过程中的某一点进行拦截调用【mybatis插件又称mybatis拦截器】。
  • 默认情况下，MyBatis 允许使用插件来拦截的方法如下：
    • 【执行器拦截】：Executor (update, query, flushStatements, commit, rollback, getTransaction, close, isClosed)
    • 【设置参数时拦截】：ParameterHandler (getParameterObject, setParameters)
    • 【处理结果集时拦截】：ResultSetHandler (handleResultSets, handleOutputParameters)
    • 【构造sql语法时拦截】： StatementHandler (prepare, parameterize, batch, update, query)
✅拦截器示例#
  • @Intercepts({@Signature(
        type= Executor.class,//指定拦截的类型
        method = "update",//Executor中的update()方法
        args = {MappedStatement.class,Object.class})})
public class MyInterceptor implements Interceptor {
    @Override
    public Object intercept(Invocation invocation) throws Throwable {
    //拦截方法，具体业务逻辑编写的位置
        return invocation.proceed();//拦截器放行
    }

    @Override
    public Object plugin(Object target) {
    //创建target对象的代理对象,目的是将当前拦截器加入到该对象中
        return Plugin.wrap(target, this);
    }

    @Override
    public void setProperties(Properties properties) {
    //属性设置
    }
}

  • 使用场景：当要更新一条记录的时候，希望这条记录没有被别人更新

⭐实现方式#

  • 取出记录时，获取当前version

  • 更新时，带上这个version

  • 执行更新时set version = newVersion where version = oldVersion 【条件为：当前version和数据库的version对比】

    • 如果相等，就将数据库version的值更改为当前version的值
    • 如果不相等，就更新失败

  • public class FindAll extends AbstractMethod {
    /**
     * @param mapperClass mapper接口
     * @param modelClass 实体类
     * @param tableInfo 数据库表反射信息
     * @return MappedStatement
     */
    @Override
    public MappedStatement injectMappedStatement(Class<?> mapperClass, Class<?> modelClass, TableInfo tableInfo) {
        /**
         * public interface LanguageDriver
         *      SqlSource createSqlSource(Configuration configuration, String script, Class<?> parameterType);
         *
         *      创建一个{@link SqlSource}来保存从注释中读取的语句。
         *      当从类或xml文件中读取映射的语句时，在启动期间调用它。
         *
         * @param configuration MyBatis配置
         * @param script 脚本注释的内容
         * @param parameterType 输入参数类型从mapper方法获得或在parameterType xml属性中指定。可以为空。
         * @return SQL源
         */
        String sql = "select  * from " + tableInfo.getTableName();//getTableName:表名称
        SqlSource sqlSource = languageDriver.createSqlSource(configuration,sql,modelClass);


        /**
         * 查询
         * mapperClass mapper接口
         * findAll mapper接口中的方法名
         * sqlSource SQL源
         * tableInfo 数据库表反射信息
         */
        return this.addSelectMappedStatementForTable(mapperClass, "findAll", sqlSource, tableInfo);
    }
}

逻辑删除#

1️⃣概述#

  • 开发系统时，有时候在实现功能时，删除操作需要实现逻辑删除，所谓逻辑删除就是将数据标记为删除，而并非真正的物理删除（非DELETE操作）【如用户订单等等】，查询时需要携带状态条件，确保被标记的数据不被查询到。这样做的目的就是避免数据被真正的删除。

2️⃣快速入门#

'version';

# 逻辑未删除值(默认为 0)

  global-config:

      logic-delete-value: 1 # 逻辑已删除值(默认为 1)
