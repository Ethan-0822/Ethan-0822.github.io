⭐什么是枚举#

  • 中文的释义，指的是可以一个一个列举

  • 即在一个特定的范围内，有穷的集合，如一年有12个月，一周有7天

  • 正因为这种有穷的集合，通常在代码设计中，使用枚举类来定义词典，如是否删除的表示，y表示是，n表示否，也可以说，键值对的映射关系，类似与map集合，redis这种。


⭐Java中的枚举诞生#

✅为何需要枚举类型

  • 枚举类型是 Java 5 中增加的一个小特性，在此之前，实现枚举的方式为定义编译期常量形式。代码中使用编译期常量的方式在理解性和安全性方面存在不足，单纯的常量数值不能够直观的体现出其作为枚举实例的目的，且实际中无限的取值空间也不能很好的与理论上有限的枚举空间相匹配。

✅静态常量实现字典

  • 假定使用int类型定义一年中的四季

    • public class Season {
    public static final int SPRING = 1;
    public static final int SUMMER = 2;
    public static final int AUTUMN = 3;
    public static final int WINTER = 4;
}

    • void testSeason(int season){
	switch (season){
		case 1 :
			// 春天...
			break;
		case 2 :
			// 夏天...
			break;
		case 3 :
			// 秋天...
			break;
		case 4 :
			// 冬天...
			break;
	}
}

    • System.out.println(Season.SPRING);
System.out.println(Season.SUMMER);

testSeason(1);
testSeason(Season.SPRING)
testSeason(-1);

  • 可以看到没有办法通知或者限制调用者需要传递的参数，只要是int值都是合法数据，这个算不上一个好的设计


✅改进静态常量实现字典

  • 改进上述例子，就可以让一个普通类达到枚举一样效果（这也是枚举反编译后的效果，下文有详述）

    • 将上述的int类型常量修改为Season类型，并将类声明为final防止被继承。构造函数声明为private，防止外部实例化。

    • public final class Season {
    public static final Season SPRING = new Season();
    public static final Season SUMMER = new Season();
    public static final Season AUTUMN = new Season();
    public static final Season WINTER = new Season();

    private Season(){
    }
}

    • void testSeason(Season season){
// ...
}

    • testSeason(Season.SPRING);
testSeason(Season.SUMMER);

testSeason(1);// 编译报错
testSeason("SPRING");// 编译报错

  • 可以看到没有通知调用者情况下，调用者只能传递已经定义好的Season类型，最多传递一个null，否则将无法通过编译，这也间接限制了调用者传递的数据类型


✅枚举实现字典

  • 例如：一个季节枚举即可表示一年中的四季

    • public enum SeasonEnum {
    SPRING,
    SUMMER,
    AUTUMN,
    WINTER;
}

    • void testSeason2(SeasonEnum season){
	switch (season){
		case SPRING :
			// 春天...
			break;
		case SUMMER :
			// 夏天...
			break;
		case AUTUMN :
			// 秋天...
			break;
		case WINTER :
			// 冬天...
			break;
	}
}

    • testSeason(Season.SPRING);
testSeason(Season.SUMMER);

testSeason(1);// 编译报错	
testSeason("SPRING");// 编译报错

    • 可以看到，枚举类型的实例摆脱了数值的限制，能够以实例自身表达其作用和目的


⭐定义枚举#

✅规则

  1. 使用关键字Enum定义枚举
  2. 枚举值需要定义在枚举的构造方法，成员变量之前，枚举值以;分号结束
    • 这里的枚举值，和Java语法要求必须先定义enum实例是同一个意思，实际上通过反编译来看就是一个 static final 构造方法
  3. 枚举的构造方法
    • 可以使用lombok的无参构造方法@NoArgsConstructor和全参构造构造方法@AllArgsConstructor
    • 但是不能使用lombok的@Data，原因可能在编译的时候，枚举值编译成static final，无法再使用@Data的setter方法
  4. 命名（阿里巴巴Java开发手册1.4.0）
  • 需要注意的只有以上4点，其余部分和普通类一样，可以在枚举中定义普通的方法，接口，重写方法等等

✅示例

  • // 最简单的枚举
public enum SeasonEnum {
    SPRING,
    SUMMER,
    AUTUMN,
    WINTER;
} 

  • // 接口
public interface Sports {
    void swim();
}

// 带参的枚举，实现接口并重写toString方法
@NoArgsConstructor
@AllArgsConstructor
public enum SeasonEnum implements Sports{
    SPRING("1","春天"),
    SUMMER("2","夏天"),
    AUTUMN("3","秋天"),
    WINTER("4","冬天");


    @Getter
    private String index;

    @Getter
    private String description;

    @Override
    public String toString() {
        return "SeasonEnum{" +
                "description='" + description + '\'' +
                ", index='" + index + '\'' +
                '}';
    }

    public void printSeason(){
        System.out.println("该枚举值的中文描述为：" +this.description);
    }

    @Override
    public void swim() {
        System.out.println("夏天最好运动的方式是游泳...");
    }
}

⭐反编译#

✅命令反编译

  • 使用javap -c SeasonEnum.class 命令反编译上述SeasonEnum枚举的字节码文件

✅还原成普通类

  • 根据字节码的图片和其他博文，尽量还原SeasonEnum枚举的普通类形式

    • 在第一张图片的黄色框中可以看到，SeasonEnum是继承了Enum的，而在编码时期是无法手动继承，只能使用关键字实现这一继承

    • 从第三张图片的static方法观察到，SeasonEnum内部声明的多个实例，其构造过程是编译器在静态代码库中，调用 SeasonEnum 自身和父类的构造函数完成定义的

    • 还原的过程省略了如toString等方法，可得如下普通类

    • public final class SeasonEnum extends java.lang.Enum {

    // 枚举值
    public static final SeasonEnum SPRING;
    public static final SeasonEnum SUMMER;
    public static final SeasonEnum AUTUMN;
    public static final SeasonEnum WINTER;

    // 枚举值数组
    private static final SeasonEnum[] $VALUES;

    // 定义的成员变量
    private String index;
    private String description;

    // 获取枚举数组
    public static SeasonEnum[] values() {
        return $VALUES.clone();
    }

    // 根据枚举值的变量名获取枚举值
    public static SeasonEnum valueOf(String name) {
        return Enum.valueOf(SeasonEnum.class, name);
    }

    // 带参构造方法
    private SeasonEnum(String name, int ordinal, String index, String description)
    {
        super(name, ordinal);
        this.index = index;
        this.description = description;
    }

    // 初始化
    static
    {
        SPRING = new SeasonEnum("SPRING", 0, "1", "春天");
        SUMMER = new SeasonEnum("SUMMER", 1, "2", "夏天");
        AUTUMN = new SeasonEnum("AUTUMN", 2, "3", "秋天");
        WINTER = new SeasonEnum("WINTER", 3, "4", "冬天");
        $VALUES = (new SeasonEnum[] {
                SPRING, SUMMER, AUTUMN, WINTER
        });
    }
}

✅源码分析

  • 观察 java.lang包下的Enum源代码其中定义的两个属性，及同名函数

    • public abstract class Enum<E extends Enum<E>> implements Comparable<E>, Serializable {

    private final String name;

    public final String name() {
        return name;
    }

    private final int ordinal;

    public final int ordinal() {
        return ordinal;
    }

    protected Enum(String name, int ordinal) {
        this.name = name;
        this.ordinal = ordinal;
    }
    ...  // omit some content
}

    • 从源码中可知 Enum的构造函数为 protected 级别，再从反编译的照片观察到 SeasonEnum内部声明的四个实例，发现并未显式调用Enum父类的构造函数，甚至连SeasonEnum自身的构造函数也没有显式调用。由此可知，枚举类内部定义的多个实例，其构造过程是由编译器调用构造函数来处理完成的。从Enum构造函数的注释，也可以佐证这一点


⭐枚举中的方法#

✅方法汇总

  • java.lang.Enum中的方法

  • SeasonEnum中的方法

    • 返回值方法描述
      SeasonEnum[]values()返回一个此枚举所有实例的数组

✅name()和toString()

  • name()
    • 根据枚举值来获取该枚举成员的变量名。
  • toString()
    • 返回枚举值的变量名。
  • 相同点
    • 作用相同，均可获取枚举成员的变量名
  • 不同点
    • 调用时的参数不同
    • name方法是用final修饰的不能被重写，而toString是可以被重写的

✅ordinal()

  • 该方法作用是，返回定义的枚举值的顺序，从0开始计数。

  • ordinal()方法是被设计用于基于枚举的复杂数据结构，例如EnumSet和EnumMap 。

  • 如果修改了枚举值定义的顺序，就会导致枚举值ordinal()的返回值发生变化

    • 所以一般枚举的做法就是在枚举内增加成员变量，再通过枚举值来自定义顺序，如本文使用的SeasonEnum枚举

    • SPRING("1","春天"),
SUMMER("2","夏天"),
AUTUMN("3","秋天"),
WINTER("4","冬天");

// 自定义顺序
@Getter
private String index;

✅valueOf()

  • // 上述反编译后的SeasonEnum枚举的valueOf()方法
public static SeasonEnum valueOf(String name) {
    return Enum.valueOf(SeasonEnum.class, name);
}

// Enum源码中的valueOf()方法
public static <T extends Enum<T>> T valueOf(Class<T> enumType, String name) {
    T result = enumType.enumConstantDirectory().get(name);
    if (result != null)
        return result;
    if (name == null)
        throw new NullPointerException("Name is null");
    throw new IllegalArgumentException(
        "No enum constant " + enumType.getCanonicalName() + "." + name);
}

  • 比较反编译生成的 valueOf 函数和Enum 源码中的 valueOf 函数，可以发现，编译器生成的 valueOf 函数内部调用的其实就是父类 Enum中定义的 valueOf 函数。

  • 而父类 Enum中定义的 valueOf 函数，其实调用的是 Class 类中定义的 enumConstantDirectory 函数，在该函数内对参数 enumType 是否为枚举类型进行判断

    • 当传入null时，抛出NullPointerException异常
    • 当传入不存在的枚举值时，抛出IllegalArgumentException异常
    • 当传入存在的枚举值时，返回枚举实例名称和枚举实例构成的 map 对象

✅values()

  • 该方法在java.lang.Enum中不存在

  • 观察反编译生成的 values 函数，返回的是枚举类内部声明的实例数组的 clone 对象，即返回一个所有枚举值的数组

    • public static SeasonEnum[] values() {
    return $VALUES.clone();
}

✅compareTo()

  • 该方法用来比较两个枚举值的ordinal值，即枚举值的定义序号

  • 观察Enum源码， Enum实现了Comparable接口，而 Comparable 接口中只有一个函数声明，即Enum重写compareTo函数。可以看到，先判断是不是同一个枚举类的变量，再返回当前枚举值和传递的枚举值差值。

    • public final int compareTo(E o) {
    Enum<?> other = (Enum<?>)o;
    Enum<E> self = this;
    if (self.getClass() != other.getClass() && // optimization
        self.getDeclaringClass() != other.getDeclaringClass())
        throw new ClassCastException();
    return self.ordinal - other.ordinal;
}

    • Season summer = Season.valueOf("SUMMER");
for(Season s : Season.values()){
    System.out.print(summer.name()+" compare to "+s.name());
    System.out.println(" result: "+summer.compareTo(s));
}

    • // 返回正数表示当前对象较大，0 表示大小相等，负数表示当前对象较小，差值表示声明顺序的差距大小。
SUMMER compare to SPRING result: 1
SUMMER compare to SUMMER result: 0
SUMMER compare to AUTUMN result: -1
SUMMER compare to WINTER result: -2

⭐注意事项#

✅阿里巴巴开发手册关于枚举建议（1.4.0）


✅二方库释义

一方库指的是本项目中的依赖

二方库指的是公司内部其他项目提供的依赖

三方库指的是其他组织、公司等来自第三方的依赖


✅为什么不建议接口的返回值包含枚举类型呢？

  • 假设有一个二方库，A系统和B系统都依赖了该二方库，同时B系统通过RPC远程调用A系统的一个接口（该接口中返回值中有一个参数类型是枚举类型）

    • // 二方库中的枚举
public enum AType{
    P_T,
    A_B
}

    • // A系统
public interface AService {
    public AResponse doSth(ARequest aRequest);
}

public Class AResponse{
    private Boolean success;
    // 返回值包括了枚举
    private AType aType;
}

    • // B系统
public class BService {
    @Autowired
    AService aService;

    public void doSth(){
        ARequest aRequest = new ARequest();
        AResponse aResponse = aFacadeService.doSth(aRequest);
        AType aType = aResponse.getAType();
    }

}

  • 当两个使用同一个枚举的时候是不会出现问题的，即在原来的二方库上可以继续使用。但在某天，二方库中的枚举进行了升级，A系统也跟着升级，而B系统并没有跟着升级，即

    • // 二方库中的枚举，也是A系统的枚举
public enum AType{
    P_T,
    A_B,
    P_M
}

// B系统的枚举
public enum AType{
    P_T,
    A_B,
}

  • 这种情况下，在B系统通过RPC调用A系统的时候，如果A系统返回的AResponse中的aType的类型为新增的P_M时候，B系统就会无法解析。一般在这种时候，RPC框架就会发生反序列化异常。导致程序被中断。


✅RPC反序列化枚举异常分析

  • RPC框架大多数会采用JSON的格式进行数据传输，也就是客户端会将返回值序列化成JSON字符串，而服务端会再将JSON字符串反序列化成一个Java对象。
  • 而JSON在反序列化的过程中，对于一个枚举类型，会尝试调用对应的枚举类的valueOf方法来获取到对应的枚举。
  • 如果从枚举类中找不到对应的枚举项的时候，就会抛出IllegalArgumentException，这也是为什么阿里巴巴不推荐在接口的返回值中包含二方库枚举值

⭐小结#

  • 总的来说，枚举在代码设计可以体现为，语义化地代表词典并作为一个独立的引用类型存在。

  • 如果担心枚举使用不当，可以使用字符串常量来代表字典，这个是最常见的一刀切方式。例如上述中RPC反序列化异常的问题，如果使用字符串代替RPC接口中的枚举，那么B系统只需要传递字符串就行，至于传递的值的合法性，只需要在A系统内进行校验就可以了。

  • 但是笔者觉得这种投鼠忌器的方式不太好，尽管枚举可能会带来扩展兼容性的问题，我们应当做的是避免出现扩展兼容性的问题（即按照阿里巴巴推荐方式）。对于单体项目或者需要快速迭代的项目，枚举便利性收益是大于兼容性的。
